什么是路由

前端路由

根据不同的url切换组件，因为app是浏览器上跑的，所以在web上切换页面要刷新，影响用户体验，所以提出了前端路由，通过前端的路由映射关系来匹配，不会再发起请求

后端路由

根据用户的请求返回不同的内容，由后端直接把页面返回，如果访问人数过多，容易给服务器造成很大的压力

React路由 V5版本

官网： https://reacttraining.com/react-router/web/guides/quick-start

中文网： https://react-router.docschina.org/

yarn add react-router-dom

下载完成后启动项目，或者使用npm也可以下载

导入

在index.js路口文件导入

import { BrowserRouter as Router } from react-router-dom

路由有两种模式：

​ hash模式（#） HashRouter 通过监听不同的hash值来改变路由 推荐使用

​ 历史记录模式（history） BrowserRouter 刷新需要重新配置请求

然后利用Router将包裹

<Router>
   <App /> 
</Router>

注意： 路由包裹了App，内部通过context向App的子孙组件去传值，关于context这里不提，可以看另一篇博客React的context

然后在App里面将两个组件引入，然后再引入路由，创建无状态组件404页面，

Switch, Route, NavLink, Redirect, Nav 都是react-router-dom 中的， 注意： exact 表示精确匹配，只有与路径完全一致才会匹配，NavLink和Nav 的区别是前面点击可以加样式，Switch 表示只会显示一个路由，Redirect表示重定向，输入 / 的时候重定向到 /home

<Route path="路径" component={组件} />
<Link to="路径">点击名<Link>
<Redirect from="/" to="/test" exact />

完整代码如下

import React from 'react';
import './App.css';
import Home from "./components/home/index"
import Two from "./components/two/Two"
import { Switch, Route, NavLink, Redirect } from "react-router-dom"

let notFind = () => {
  return (
    <div>404啊</div>
  )
}

function App() {
  return (
    <div className="App">
      <NavLink to="/home">home</NavLink>
      <NavLink to="/two">two</NavLink>

      <Switch>
        <Route path="/home" component={Home} />
        <Route path="/two" component={Two} />
        <Redirect from="/" to="/home" exact />
        <Route component={notFind} />
      </Switch>
    </div>
  );
}

export default App;

嵌套路由

有了上面的基础，我们再来写一个多级的嵌套路由,创建一个One 的文件夹，创建OneA和OneB组件，然后在Home中引入，路由使用方式和上面的一样，就是注意重定向问题，直接上码

这个是改完之后的home组件,OneA组件和OneB组件省略

import React, { Component } from 'react'
import { Redirect, Route, Switch, NavLink } from "react-router-dom"
import OneA from "../one/OneA"
import OneB from "../one/OneB"
const oneFind = () => {
  return <div>404</div>
}
export default class index extends Component {
  render() {
    return (
      <div>
        <NavLink to="/home/onea">OneA</NavLink>
        <NavLink to="/home/oneb">OneB</NavLink>

        <Switch>
          <Route path="/home/onea" component={OneA}></Route>
          <Route path="/home/oneb" component={OneB}></Route>
          <Redirect from="/home" to="/home/onea"></Redirect>
          <Route component={oneFind}></Route>
        </Switch>
      </div>
    )
  }
}

动态路由

我们在App,js 中写路由 通过 在斜杠后面加：实现动态路由,传值过去，types就是你要传的值

 <Route path="/list/:types" component={List} />
//APP
class App extends React.Component {
  render() {
    return (
      <div>
        <NavLink to="/list/users">user</NavLink>
        <NavLink to="/list/goods">good</NavLink>

        <Route path="/list/:types" component={List} />
      </div>
    )
  }
}

然后我们创建一个路由组件，通过this.props.match.params.types 来取得传过来的types的值，注意：App.js中要使用withRouter ，若不然没有match这个值，然后在路由组件中要注意判断传过来的值是否与上一次的相等，否则容易造成死循环，一直监听到值的更改，一直发请求

import React, { Component } from 'react'

export default class ccc extends Component {
  constructor(props) {
    super(props)
    this.state = {
      list: [],
      type: ""
    }
  }

  componentDidMount() {
    this.getData()
  }
  componentDidUpdate() {   // 在这里监听传入的types的值的改变
    this.getData()
  }

  getData() {
    let types = this.props.match.params.types
    if (this.state.type !== types) {  // 注意，这里一定要判断传入的值是否与之前的值相等
      fetch("http://rap2api.taobao.org/app/mock/226329/api/" + types).then(res => res.json()).then(res => {
        this.setState({
          list: res.data.list,
          type: types   //更改types的值
        })
      })
    }
  }

  render() {
    const { list } = this.state
    return (
      <div>
        {
          list.map(item => {
            return <li key={item.id}> {item.id} {item.mz}</li>
          })
        }
      </div>
    )
  }
}

路由鉴权

当用户要跳转到two页面的时候，判断有没有登录（即判断有没有cookie）如果有，则登录，如果没有，则重定向到登录页面

所以，利用了 Route 的另一种形式，render，让Route 可以实现传参

  <Route path="/Two" render={(props) => {
            return sessionStorage.getItem("user") ? <Two /> : <Redirect to="/login" />
       }}></Route>

将其封装为一个组件 MyRouter, 通过父组件给子组件传参，传入组件和path，下面式封装

import React, { Component } from 'react'
import { Route, Redirect } from "react-router-dom"
export default class MyRouter extends Component {
  render() {
    const { path, component: Com } = this.props
    return (
      <div>
        <Route path={path} render={(props) => {
          // return sessionStorage.getItem("user") ? <Com {...props} /> : <Redirect to="/login"  />
          return sessionStorage.getItem("user") ? <Com {...props} /> : <Redirect to={{ pathname: "/login", state: { path } }} />
        }}></Route>
      </div>
    )
  }
}

import React, { Component } from 'react'

export default class login extends Component {
  getUser = (e) => {
    if (e.keyCode === 13) {
      if (this.node.value === "admin") {
        sessionStorage.setItem("user", this.node.value);
        // this.props.history.push("/two")
        if (this.props.location.state) { //判断用户是否直接登陆进来的
          //tage 就是应该登陆跳转到的路由的路径
          var tage = this.props.location.state.path;
          this.props.history.push(tage)
        } else {
          //用户直接进入的login组件
          //如果用户直接登陆，应该跳转到home页面
          this.props.history.push("/two")
        }

      }
    }
  }
  render() {
    return (
      <div>
        <input type="text" ref={node => this.node = node} onKeyUp={this.getUser} />
      </div>
    )
  }
}
